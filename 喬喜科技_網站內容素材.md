## 三、問題定義（Why / Problem）

### 標題
> 關鍵知識，正在隨人離開

### 內文

想像一下，一個在工廠待了 20 年的老師傅，下個月就要退休了。他腦中那些處理異常狀況的判斷、那些沒寫在 SOP 裡的 know-how，就這樣跟著他走了。

農會信用部的資深主任，看到「種稻、申貸八百萬」就知道不對勁。這種風控直覺從未被寫進任何文件。工廠的老師傅，聽聲辨位就知道三號軸承有問題。中小企業的管理知識，全壓在老闆一個人腦袋裡。

這些工作不能消失，但真的沒人願意做了。

「知識傳承」這件事的前提是假的——傳承假設有人要接，但現實是接的人不會出現了。

### 數據佐證（可用於視覺化呈現）

- 臺灣傳統製造業平均從業人員年齡持續上升
- 農漁會基層人員招聘長期不足
- 中小企業關鍵知識集中於少數資深員工

## 四、解決方案（What / Solution）

### 標題
> AI 駐廠顧問：讓資深員工的判斷力永遠留在公司

### 核心概念

喬喜科技的 AI 駐廠顧問，由專業顧問實際進駐合作企業，深入理解該企業的知識體系、作業流程與決策邏輯。透過 Ontology 驅動的知識工程方法，將散落在文件、歷史案例與例外處理紀錄中的經驗，結構化地萃取並建構為企業專屬的知識模型。

不是一套搜尋工具，不是一個聊天機器人。是一個真正理解「為什麼這份文件在這個情境下重要」的 AI 系統。

### 三階段架構

**第一階段：Structure（結構化）**
建立企業知識的 Ontology 骨架。師傅管什麼、工作流程、基礎知識、人與設備的關係——先把靜態的知識結構建出來。這一步用對話式引導，操作者不需要懂技術，只是在跟一個很有條理的系統聊天。

**第二階段：Capture（捕捉）**
Ontology 驅動 Scenario 自動生成，系統主動問老師傅：「如果溫度突然飆高你怎麼做？」「如果同時原料批次也有問題呢？」根據回答即時判斷哪些知識路徑還是空的，自動補洞。師傅是種子，AI 是放大器。

**第三階段：Deploy（部署）**
AI Agent 上線，在真實業務場景中提供決策支援。從影子模式（與資深員工並行驗證）到正式接手，逐步建立信任。

### 技術亮點

- **Ontology-first 方法論：** 三層知識系統（Domain Ontology → Knowledge Graph → Documents），加上動態層（Action Types → Business Rules → Decision Patterns）
- **Ontology 增強 RAG：** 查詢擴展、混合檢索、上下文豐富化、推理痕跡——不只找到正確的文件，更理解文件在特定情境下為何重要
- **知識萃取 SOP 化：** 萃取流程設計為 idiot-proof，不需要 AI 專家駐廠，普通操作人員按流程即可完成
- **產業 Ontology 可遷移：** 同產業第二家客戶導入時間預估縮短 40-50%

## 五、產品價值（Benefits）

### 對單一客戶

**降低人才斷層風險**
資深員工退休或離職時，專業判斷與處理經驗不再隨人流失。以農會信用部為例，資深授信主任看到「種稻配八百萬」時立即警覺用途與金額不合理，這類判斷從未被寫進 SOP。AI 駐廠顧問將此類風控直覺結構化保存。預估可保留約 70-80% 的關鍵隱性知識，將人員異動對業務品質的衝擊降低約 50%。

**縮短新人上手時間**
新進人員可即時查詢歷史案例與決策邏輯。面對從未處理過的異常案例，不需要等資深同事有空才能問。預估可將新人培訓期從 6-12 個月縮短至 2-4 個月，降低約 50-60% 的教育訓練時間成本。

**提升例行作業效率**
針對重複性高的查詢、表單填寫與文件比對，預估減少約 30-40% 的人工處理時間。例如農會信用部每日的申貸文件交叉比對與資格審核，AI 可自動擷取關鍵欄位、比對歷史紀錄並標示異常。

**關鍵情境的即時決策支援**
在高風險或罕見情境中即時提供參考依據。例如基金會面臨突發性大量需求調度，系統根據過往調度紀錄與資深調度員的決策模式，在數秒內提出建議方案，將原本 30 分鐘以上的緊急決策壓縮至 5 分鐘內。

### 對市場（飛輪效應）

當喬喜在一個產業累積越來越多資深專家的判斷邏輯，手上握的就不再是單一客戶的資產，而是整個產業的集體決策智慧。

- **新客戶導入成本趨近於零：** 不用從零建 Ontology，已有產業知識圖譜，只需校準特殊變數
- **產品越用越強：** 每一次決策、每一個專家的判斷被記錄，因果圖就更密更準——純粹的網路效應
- **訓練產業專用 AI Agent 的獨有資料：** LLM 很會說話但不懂特定產業的因果邏輯。結構化的因果圖加真實 Decision Trace，是別人沒有的東西

## 八、技術架構（Technology）

### 知識系統分層

**語義層（Semantic Layer）**
1. Domain Ontology — 產業領域的概念定義與關係結構
2. Knowledge Graph — 具體實例的知識網絡
3. Documents — 原始文件、歷史案例、SOP

**動態層（Kinetic Layer）**
4. Action Types — 動作的前置條件、觸發規則、影響範圍
5. Business Rules / Constraints — 不能做什麼、順序限制、閾值
6. Decision Patterns — 資深者的判斷、排序、取捨邏輯

### RAG 技術棧（以農會信用部系統為例）

- Guard LLM 路由（GPT-4o-mini）
- 混合搜尋（BM25 + Vector，動態 RRF 加權）
- Parent Section Retrieval
- LLM Reranking
- 結構化 Pydantic 輸出 + CoT 推理
- 智慧表單自動填寫（PyMuPDF / Pillow）
- Canvas-based 管理端標記 UI
- Human-in-the-loop 黃金 QA 回饋迴路

### 系統架構

- 前端：React + TypeScript
- 後端：FastAPI + LangChain + PostgreSQL
- 認證：JWT / RBAC
- 即時通訊：WebSocket / SSE
- 資安：OWASP LLM Top 10 風險評估
